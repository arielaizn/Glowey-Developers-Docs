# Files API

The Files API manages artifacts generated during SuperGlow pipelines — images, videos, scripts, and other media. Files are stored in a secure Supabase bucket and scoped to the authenticated user.

## GET /api/superglow/files

List files for the authenticated user, optionally filtered by agent/conversation ID.

### Authentication

Bearer token required.

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `agentId` | string | No | Filter by conversation/agent ID. If omitted, returns all user files. |

### Response

```json
{
  "files": [
    {
      "name": "1716123456789-quantum_visualization.jpg",
      "id": "1716123456789-quantum_visualization.jpg",
      "updated_at": "2025-05-19T10:30:00.000Z",
      "created_at": "2025-05-19T10:30:00.000Z",
      "last_accessed_at": null,
      "metadata": {
        "eTag": "abc123xyz789",
        "size": 2097152
      }
    }
  ]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | File name (includes timestamp prefix for uniqueness) |
| `id` | string | File ID (same as name) |
| `updated_at` | ISO 8601 | Last update time |
| `created_at` | ISO 8601 | Creation time |
| `metadata` | object | Size and ETag from storage backend |

### Errors

- `401` – Unauthorized
- `429` – Rate limited
- `500` – Server error

### Example Request

```bash
# List all user files
curl "https://glowey.app/api/superglow/files" \
  -H "Authorization: Bearer glow_sk_YOUR_TOKEN_HERE"

# List files for a specific conversation
curl "https://glowey.app/api/superglow/files?agentId=conv_19a8b7c6d5e4f" \
  -H "Authorization: Bearer glow_sk_YOUR_TOKEN_HERE"
```

## POST /api/superglow/files

Upload a file. Files are automatically prefixed with user ID and conversation ID for organization.

### Authentication

Bearer token required.

### Request Format

Multipart form data with the following fields:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `file` | File | Yes | File to upload (any type) |
| `agentId` | string | No | Conversation ID. If omitted, defaults to `default` |

### Response

```json
{
  "path": "user_uuid/conv_19a8b7c6d5e4f/1716123456789-my_image.jpg"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `path` | string | Storage path (used for deletion later) |

Returns status 201 on success.

### File Storage Path

Files are stored in the bucket at:
```
superglow-artifacts/
├── {user_id}/
│   ├── {agentId or "default"}/
│   │   ├── {timestamp}-{filename}
│   │   └── ...
│   └── ...
```

This ensures files are automatically scoped to the user and conversation.

### Errors

- `400` – No file provided
- `401` – Unauthorized
- `429` – Rate limited
- `500` – Server error

### Example Request (cURL)

```bash
curl -X POST https://glowey.app/api/superglow/files \
  -H "Authorization: Bearer glow_sk_YOUR_TOKEN_HERE" \
  -F "file=@/path/to/image.jpg" \
  -F "agentId=conv_19a8b7c6d5e4f"
```

### Example Request (Node.js)

```javascript
const formData = new FormData();
formData.append("file", fileBlob, "image.jpg");
formData.append("agentId", "conv_19a8b7c6d5e4f");

const response = await fetch(
  "https://glowey.app/api/superglow/files",
  {
    method: "POST",
    headers: {
      "Authorization": "Bearer glow_sk_YOUR_TOKEN_HERE",
    },
    body: formData,
  }
);

const data = await response.json();
console.log("File stored at:", data.path);
```

## DELETE /api/superglow/files

Delete a file by its storage path.

### Authentication

Bearer token required.

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Storage path returned from POST (security check ensures ownership) |

### Response

```json
{
  "ok": true
}
```

### Security

The endpoint verifies that the `path` belongs to the authenticated user (starts with `{user_id}/`). Attempting to delete another user's file returns 403 Forbidden.

### Errors

- `400` – Missing `path`
- `401` – Unauthorized
- `403` – Forbidden (path belongs to another user)
- `429` – Rate limited
- `500` – Server error

### Example Request

```bash
curl -X DELETE "https://glowey.app/api/superglow/files?path=user_uuid/conv_19a8b7c6d5e4f/1716123456789-image.jpg" \
  -H "Authorization: Bearer glow_sk_YOUR_TOKEN_HERE"
```

## Accessing Files

Once uploaded or generated, files can be accessed directly via Supabase storage URLs:

```
https://storage.glowey.app/superglow-artifacts/{path}
```

Example:
```
https://storage.glowey.app/superglow-artifacts/user_uuid/conv_19a8b7c6d5e4f/1716123456789-image.jpg
```

These URLs are signed and scoped to the authenticated session.

## Use Cases

### Store Generated Assets

After Hermes completes a generation run, upload the final video or images:

```bash
# Upload final video
curl -X POST https://glowey.app/api/superglow/files \
  -H "Authorization: Bearer glow_sk_YOUR_TOKEN_HERE" \
  -F "file=@final_video.mp4" \
  -F "agentId=conv_19a8b7c6d5e4f"

# Response: path = "user_uuid/conv_19a8b7c6d5e4f/1716123456789-final_video.mp4"
```

### Retrieve Conversation Assets

List all files for a conversation:

```bash
curl "https://glowey.app/api/superglow/files?agentId=conv_19a8b7c6d5e4f" \
  -H "Authorization: Bearer glow_sk_YOUR_TOKEN_HERE"
```

### Cleanup Old Assets

Delete intermediate files after the final composite is ready:

```bash
curl -X DELETE "https://glowey.app/api/superglow/files?path=user_uuid/conv_19a8b7c6d5e4f/1716123456789-intermediate.jpg" \
  -H "Authorization: Bearer glow_sk_YOUR_TOKEN_HERE"
```

## Storage Limits

- **Per file**: Up to 2 GB (typical for videos)
- **Per user**: Contact support for quota details
- **Retention**: Files are retained for the lifetime of the conversation unless explicitly deleted

For bulk deletions or quota increases, contact Glowey support.
